# MILESTONE 20: SINGLE PRODUCTION VOICE PATH

## Goal

Collapse parallel voice stacks into one production path:

- OpenClaw owns speech shell and presence.
- JARVIS owns consciousness, continuity, relationship mode, pushback, and time framing.

## Production Contract

### JARVIS authority

- continuity envelope and contract hash
- user model and memory authority
- relationship mode (`equal` / `strategist` / `butler`)
- hypothesis transparency + pushback behavior
- risk and approval governance

### OpenClaw authority

- Talk Mode loop (listen -> send -> speak)
- wake/ptt shell behavior
- transport and channel/session presence
- STT/TTS provider orchestration

## Scope

1. Add a production OpenClaw voice config template.
2. Add a bootstrap script to:
   - validate Node/OpenClaw runtime requirements
   - install local STT dependency (`whisper-cpp`) when requested
   - merge production voice template into `~/.openclaw/openclaw.json`
3. Keep local `jarvis_voice_chat.py` as fallback harness, not primary production shell.
4. Keep reply orchestration output mode-aware voice directives for OpenClaw rendering.

## Required Outputs

- `configs/openclaw.voice.production.template.json`
- `scripts/setup_openclaw_voice_prod.sh`
- `exports/openclaw.voice.production.preview.json` (generated by script)

## Acceptance Criteria

- One command can prepare production voice config:
  - `scripts/setup_openclaw_voice_prod.sh --install-prereqs --apply`
- Script reports actionable blockers (Node version, missing `openclaw`, missing `whisper-cli`).
- Production template merges without destroying unrelated user config.
- Voice replies retain JARVIS mode selection while exposing OpenClaw speech directive metadata.
